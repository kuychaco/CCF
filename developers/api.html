std::optional<std::vector<uint8_t>> process(std::shared_ptr<enclave::RpcContext> ctx)¶

Process a serialised command with the associated RPC context

If an RPC that requires writing to the kv store is processed on a backup, the serialised RPC is forwarded to the current network primary.

Return

nullopt if the result is pending (to be forwarded, or still to-be-executed by consensus), else the response (may contain error)

Parameters

  • ctx: Context for this RPC
